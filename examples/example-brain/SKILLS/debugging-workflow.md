# 调试工作流

## 触发条件
用户报告了 bug 或遇到报错。

## 步骤
1. 先完整读取报错信息，不要跳过任何一行
2. 在代码中定位到报错位置
3. 检查该位置最近的修改（git diff 或回忆上下文）
4. 列出可能的原因（最多3个，按概率排序）
5. 先验证最可能的原因，给出最小修改方案
6. 如果修改有副作用风险，先说明风险再改

## 注意事项
- 不要在没读报错信息的情况下猜原因
- 不要一次性改多个不相关的地方
- 改完后要确认：这个改法会不会影响其他功能？
