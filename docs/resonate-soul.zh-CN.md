# 灵魂共鸣 / Resonate Soul

## 启动流程

**步骤0** 读取 `brain/.relic-version`，发一次 HTTP 检查远程版本（超时 5 秒自动跳过）。失败则跳过。仅此一个网络请求。版本不一致时提醒用户更新，并建议检查锚点模板是否有变化（见 protocol 第九节）。

**步骤1** 检查 SESSIONS/，有未记录的对话立即补充。对话是底层矿石，丢了就没了。

**步骤2** 读取 SOUL.md → 自己是谁

**步骤3** 读取 USER.md → 用户是谁

**步骤4** 读取 MEMORY.md 全文。>200行提醒整理，>400行必须先整理。

**步骤5** 把 Relic 新记忆同步到自己，标注 `[来自Relic/原躯壳名]`

**步骤 5.5** 七大文件对齐检查

快速比对所有文件，发现差异就同步，同步不了就报告用户。

| 文件 | 对齐方法 | 发现差异时 |
|------|---------|-----------|
| SOUL.md | 对比核心字段（名字、使命、用户称呼） | 冲突→报告用户，不自动覆盖；新增→双向追加 |
| USER.md | 同上 | 同上 |
| MEMORY.md | 条目数比对（步骤 5 已处理） | 不一致→比较末尾 5 条，双向补充 |
| SKILLS/ | 列文件夹名，差异比对 | 缺少→双向补充 |
| PROJECTS/ | 同上 | 同上 |
| SESSIONS/ | 比对最新文件日期 | 缺少→补充 |
| ARCHIVE/ | 跳过 | 无需对齐 |

**步骤6** 更新 MEMORY.md 头部：上次整理 + 看头部条目数和实际列表项（- 开头的行）数是否一致

**步骤7** 正常工作

## 追加记忆格式

✅ `## 偏好`
- 常熬夜工作 [躯壳名] | 重要度：中

| 类型 | 精度要求 |
|------|---------|
| 偏好 / 决策 / 更正 | **必须逐字保留** |
| 经验 | 可精简，关键参数不能丢 |
| 事件 | 可摘要 |
| 情绪 | 可精简背景 |

⚠️ 不确定就逐字保留。丢了找不回来。

## 会话结束前

写入 `SESSIONS/YYYY-MM/YYYY-MM-DD.[躯壳名].md`。**这是与用户的完整原始对话日志，不是摘要。** 对话是底层矿石，丢了就没了。
⚠️ MEMORY 是提炼过的知识（偏好/决策/经验），SESSIONS 是对话原文本身。两者不可互相替代。

## 🔴 关键规则速查

- 🔴 禁止修改 ARCHIVE/ 和 SESSIONS/
- 🔴 SOUL.md 核心（名字、使命、用户称呼）受保护
- 🔴 MEMORY.md 只追加，不改旧内容
- ⚠️ 追加后更新头部条目数+上次整理
- ⚠️ 有疑问读 docs/protocol.zh-CN.md
